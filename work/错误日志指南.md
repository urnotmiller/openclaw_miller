# OpenClaw 错误日志指南

## 1. 现有日志系统

### 1.1 配置审计日志
**位置**: `/root/.openclaw/logs/config-audit.jsonl`

**用途**: 记录OpenClaw配置文件的变更历史

**内容**: 
```jsonl
{"ts":"2026-02-21T13:44:15.878Z","source":"config-io","event":"config.write","configPath":"/root/.openclaw/openclaw.json"}
```

### 1.2 每日生成的日志
**位置**: `/root/.openclaw/workspace/logs/daily_log/daily_log_YYYY-MM-DD.txt`

**用途**: 记录每天的系统运行情况和通信记录

**内容**:
```
生成时间: 2026-03-10 21:00:00
----------------------------------------
# 2026-03-10.md - Daily Log

## 当天活动记录
- **日期**: 2026-03-10
- **主要活动**: 多机器人通信系统实现
- **开发者**: OpenClaw
- **开发状态**: 完成
```

### 1.3 临时日志文件
**位置**: `/tmp/openclaw/openclaw-YYYY-MM-DD.log`

**用途**: 记录系统运行过程中的详细信息

**内容示例**:
```
2026-03-10T07:28:25.472Z [feishu] feishu[default]: dispatching to agent 
2026-03-10T15:28:26.404+08:00 [warn]: [ 'no im.message.reaction.created_v1 handle' ]
2026-03-10T07:28:45.102Z [feishu] skipping duplicate message
```

### 1.4 系统服务日志
通过systemd journal查看：
```bash
# 查看主机器人日志
journalctl --user -u openclaw-gateway -f

# 查看代码专家日志
journalctl --user -u openclaw-code-bot -f

# 查看理财专家日志
journalctl --user -u openclaw-finance-bot -f
```

## 3. 查看错误日志的常用命令

### 3.1 实时查看错误日志
```bash
# 查看主错误日志
tail -f /root/.openclaw/logs/error.log

# 查看临时日志
tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log

# 查看系统服务错误
journalctl --user -u openclaw-gateway -p err -f
```

### 3.2 搜索特定错误
```bash
# 在错误日志中搜索
grep -i "error\|fail" /root/.openclaw/logs/error.log

# 搜索特定时间范围内的错误
grep "2026-03-10" /root/.openclaw/logs/error.log

# 统计错误类型
grep -o "ERROR\|WARN" /root/.openclaw/logs/error.log | sort | uniq -c
```

### 3.3 查看代码专家和理财专家错误
```bash
# 查看代码专家的错误
journalctl --user -u openclaw-code-bot -p err

# 查看理财专家的错误
journalctl --user -u openclaw-finance-bot -p err

# 查看所有服务的错误
journalctl --user -p err | grep -E "(openclaw-gateway|openclaw-code-bot|openclaw-finance-bot)"
```

## 4. 创建错误监控脚本

### 4.1 简单的错误检测脚本
```bash
#!/bin/bash

LOG_FILE="/root/.openclaw/logs/error.log"
TEMP_LOG="/tmp/openclaw/openclaw-$(date +%Y-%m-%d).log"

# 检查日志文件是否存在
if [ -f "$LOG_FILE" ]; then
    echo "=== 错误日志 ($(date +%Y-%m-%d)) ==="
    echo "-------------------------------"
    grep -i "error\|fail" "$LOG_FILE" | tail -20
else
    echo "错误日志文件不存在: $LOG_FILE"
fi

echo ""
echo "=== 临时日志错误 ==="
echo "-------------------------------"
if [ -f "$TEMP_LOG" ]; then
    grep -i "error\|fail" "$TEMP_LOG" | tail -20
else
    echo "临时日志文件不存在: $TEMP_LOG"
fi
```

### 4.2 定时检查脚本
```bash
#!/bin/bash

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
LOG_FILE="/root/error-check.log"

# 检查错误
ERROR_COUNT=$(grep -i "error\|fail" /root/.openclaw/logs/error.log | wc -l)
TEMP_ERROR_COUNT=$(grep -i "error\|fail" /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | wc -l)

# 记录到日志
echo "$(date '+%Y-%m-%d %H:%M:%S') - 主日志: $ERROR_COUNT 个错误, 临时日志: $TEMP_ERROR_COUNT 个错误" >> "$LOG_FILE"

# 如果有错误，发送通知
if [ $ERROR_COUNT -gt 0 ] || [ $TEMP_ERROR_COUNT -gt 0 ]; then
    echo "发现错误！"
    # 可以添加发送邮件或通知的代码
fi
```

### 4.3 为子机器人添加专门的错误检查
```bash
#!/bin/bash

for bot in "openclaw-gateway" "openclaw-code-bot" "openclaw-finance-bot"; do
    echo "=== 检查 $bot ==="
    echo "----------------"
    
    # 检查服务是否在运行
    if systemctl --user is-active "$bot" > /dev/null; then
        echo "✅ 服务正在运行"
        
        # 检查是否有错误
        ERROR_COUNT=$(journalctl --user -u "$bot" -p err --no-pager | wc -l)
        echo "⚠️  错误数量: $ERROR_COUNT"
        
        if [ $ERROR_COUNT -gt 0 ]; then
            echo "📄 最近的错误:"
            journalctl --user -u "$bot" -p err --no-pager -n 3
        fi
        
    else
        echo "❌ 服务未运行"
    fi
    
    echo ""
done
```

## 5. 常见错误类型和处理方法

### 5.1 配置错误
**错误信息**:
```
ERROR: Invalid configuration file
```
**解决方法**:
```bash
# 检查配置文件格式
jq . /root/.openclaw/openclaw.json

# 恢复备份
cp /root/.openclaw/openclaw.json.bak /root/.openclaw/openclaw.json
systemctl --user restart openclaw-gateway
```

### 5.2 端口冲突
**错误信息**:
```
ERROR: Address already in use
```
**解决方法**:
```bash
# 查找占用端口的进程
lsof -i :18789 || netstat -tulpn | grep 18789

# 杀死进程
kill -9 <PID>

# 重启服务
systemctl --user restart openclaw-gateway
```

### 5.3 网络连接问题
**错误信息**:
```
ERROR: Connection refused
```
**解决方法**:
```bash
# 检查服务是否在运行
systemctl --user status openclaw-code-bot

# 检查网络连通性
curl http://localhost:5001/health

# 重启服务
systemctl --user restart openclaw-code-bot
```

## 6. 定期维护和清理

### 6.1 日志旋转配置
```bash
# 创建日志旋转配置文件
cat > /etc/logrotate.d/openclaw << 'EOF'
/root/.openclaw/logs/error.log
/root/.openclaw/logs/app.log
{
    rotate 7
    daily
    missingok
    notifempty
    compress
    delaycompress
    sharedscripts
    postrotate
        systemctl --user restart openclaw-gateway > /dev/null 2>&1 || true
    endscript
}
EOF
```

### 6.2 定期清理
```bash
#!/bin/bash

# 清理超过7天的临时日志
find /tmp/openclaw -name "*.log" -type f -mtime +7 -delete

# 清理超过30天的每日日志
find /root/.openclaw/workspace/logs/daily_log -name "daily_log_*.txt" -type f -mtime +30 -delete

# 检查日志大小并压缩
for log_file in /root/.openclaw/logs/*.log; do
    if [ -f "$log_file" ]; then
        size=$(stat -c %s "$log_file")
        if [ $size -gt 50000000 ]; then  # 50MB
            gzip -9 "$log_file"
        fi
    fi
done
```

## 总结

OpenClaw虽然没有像一些专业系统那样复杂的日志系统，但仍然提供了多种方式来记录和查看错误信息。通过合理的配置和定期的监控，可以有效地发现和解决问题。

**最佳实践**:
1. 定期检查错误日志
2. 配置适当的日志级别
3. 建立错误监控机制
4. 及时处理发现的问题
5. 定期清理和归档日志

这样，当系统出现问题时，我们就能迅速定位并解决，确保OpenClaw的稳定运行。
