## 测试用例

### 1. 天气查询测试

#### TC01: 有效城市天气查询（北京、上海等）
- **用例ID**：TC01
- **测试目标**：验证有效城市可以正常查询天气
- **前置条件**：后端服务已启动，端口 3000 可用
- **测试步骤**：
  1. 调用 GET /api/weather/北京
  2. 调用 GET /api/weather/上海
  3. 调用 GET /api/weather/广州
- **预期结果**：
  - 返回状态码 200
  - 响应包含 `data` 字段
  - `data.current` 存在且包含温度、天气描述
- **优先级**：P0

#### TC02: 无效城市返回错误
- **用例ID**：TC02
- **测试目标**：验证不支持的城市返回友好错误
- **前置条件**：后端服务已启动
- **测试步骤**：
  1. 调用 GET /api/weather/不存在城市
- **预期结果**：
  - 返回错误提示（code !== 0 或 message 包含"不存在"）
- **优先级**：P1

#### TC03: 三日预报数据完整性（yesterday/today/tomorrow）
- **用例ID**：TC03
- **测试目标**：验证三日预报数据结构完整
- **前置条件**：后端服务已启动
- **测试步骤**：
  1. 调用 GET /api/weather/北京
  2. 检查返回数据是否包含 yesterday、today、tomorrow 三个字段
- **预期结果**：
  - `data.yesterday` 存在
  - `data.today` 存在
  - `data.tomorrow` 存在
- **优先级**：P0

### 2. 穿着推荐测试

#### TC04: 穿搭卡片内容非空
- **用例ID**：TC04
- **测试目标**：验证穿搭推荐卡片显示有效内容
- **前置条件**：页面已加载，可查询天气
- **测试步骤**：
  1. 查询任意城市天气（如北京）
  2. 检查穿搭推荐卡片是否显示
- **预期结果**：
  - 穿搭卡片有内容显示（非空白）
  - 包含服装类型或搭配建议
- **优先级**：P1

#### TC05: 换穿搭按钮点击后内容变化
- **用例ID**：TC05
- **测试目标**：验证换穿搭按钮功能正常
- **前置条件**：穿搭卡片已显示
- **测试步骤**：
  1. 记录当前穿搭内容
  2. 点击"换穿搭"按钮
  3. 记录点击后的穿搭内容
- **预期结果**：
  - 换穿搭后内容发生变化
  - 不出现连点问题（内容不应在短时间内多次变化）
- **优先级**：P1

### 4. 设置面板测试

#### TC07: 温度单位切换生效
- **用例ID**：TC07
- **测试目标**：验证温度单位可在摄氏度/华氏度间切换
- **前置条件**：设置面板可用
- **测试步骤**：
  1. 默认显示为摄氏度（°C）
  2. 切换到华氏度（°F）
  3. 验证温度数值变化
  4. 切换回摄氏度验证恢复
- **预期结果**：
  - 温度单位正确切换
  - 温度数值按对应单位转换显示
- **优先级**：P2
