## 1. 通用组件交互规范

### 1.1 Button（按钮）

#### Default（默认态）
- 背景色填充，边框圆角 12px，文字居中
- Primary: `#8DB48E` 背景，白色文字
- Secondary: 透明背景，品牌绿边框和文字

#### Hover（悬停态）
- 触发：鼠标悬停 / focus
- Primary: 背景变深 `#5C8A5C`，`transition: 150ms ease`
- Secondary: 背景 `#E8F5E9`（浅绿）
- Shadow: `shadow-sm` 轻微加深

#### Active / Pressed（按压态）
- 触发：鼠标按下 / touch start
- 视觉反馈：`transform: scale(0.97)`，阴影内陷
- 持续时间：按住期间

#### Disabled（禁用态）
- 透明度降低至 `opacity: 0.45`
- `cursor: not-allowed`
- 无 hover / active 效果

#### Loading（加载态）
- 文字替换为 Loading Spinner（12px 圆形旋转）
- 按钮宽度保持不变，防止布局抖动
- 背景色降为 `opacity: 0.7`

#### 点击反馈
- 触发时机：mousedown / touchstart
- 视觉：按钮 `scale(0.97)` → `scale(1.02)` → `scale(1)`，总计 200ms

---

### 1.2 Card（卡片）

#### Default（默认态）
- 白色背景，`shadow-sm`，圆角 16px

#### Hover（悬停态）
- `transform: translateY(-2px)`
- `box-shadow` 升级为 `shadow-md`
- `transition: 200ms ease`
- 光标 `cursor: pointer`（仅在可点击卡片上）

#### Selected / Active（选中态）
- 左侧出现 4px 宽品牌绿竖条（`border-left: 4px solid #8DB48E`）
- 背景保持白，轻微高亮

#### 卡片点击区域
- 整个卡片可点击时，整张卡片应用 click handler
- 不使用子元素嵌套多个独立点击区域（避免点击区域重叠冲突）

---

### 1.3 Chip / Tag（标签）

#### Default
- Pill 形状，`radius-full`，内边距 `4px 12px`
- 背景色语义化：绿色=推荐、橙色=提示、灰色=次要

#### Hover
- 背景色加深 10%
- `cursor: pointer`

#### Active / Selected
- 边框 1.5px 实色，背景变为极浅对应色
- 例：选中"防晒"→ 背景 `#FFF8E1`，边框 `#FFB74D`

#### 可删除标签
- 右侧带小 × 图标
- 点击 × 触发删除，回调 `onRemove(tagId)`
- × 图标 hover 变为 `#F44336`（红色）

---

### 1.4 Toast / Notice（提示条）

#### 触发方式
- 自动触发：操作成功后（如"定位成功"）
- 手动触发：网络错误、API 失败

#### 展示规则
- 出现位置：顶部 `position: fixed; top: 16px`，水平居中
- 停留时间：3000ms（可配置）
- 动画：从 `opacity: 0; translateY(-20px)` → `opacity: 1; translateY(0)`

#### 消失方式
- 定时消失（3s 后淡出）
- 手动关闭：点击右侧 × 按钮

#### 层级
- 固定 `z-index: 9999`
- 多条 Toast 垂直堆叠，间距 8px

#### 类型与颜色
| 类型 | 左侧色条 | 背景色 |
|------|---------|--------|
| Success | `#4CAF50` | `#F1F8E9` |
| Warning | `#FF9800` | `#FFF8E1` |
| Error | `#F44336` | `#FFEBEE` |
| Info | `#2196F3` | `#E3F2FD` |

---

### 1.5 Bottom Sheet（底部弹层）

#### 触发方式
- 点击城市选择、日期选择、更多设置时触发

#### 展开动画
- 从底部滑入：`translateY(100%)` → `translateY(0)`
- 时长 300ms，`ease: cubic-bezier(0.4, 0, 0.2, 1)`
- 背景遮罩：`opacity: 0` → `opacity: 1`，同步动画

#### 关闭方式
- 点击背景遮罩
- 点击顶部 × 关闭按钮
- 手指向下滑动（touch 设备）

#### 下滑关闭判定
- 滑动手势：下滑距离 > 100px 且速度 > 0.5px/ms 时关闭
- 滑动手势未达标时，弹层回弹到原位

#### 内容区
- 最大高度：`70vh`，超出部分内部滚动
- 顶部有拖拽指示条（宽 36px，高 4px，圆角 2px，灰色 `#BDBDBD`）

---

### 1.6 Modal / Dialog（模态对话框）

#### 触发方式
- 权限请求、确认操作（如"清除定位历史"）

#### 动画
- 遮罩：`opacity: 0 → 1`，200ms
- 对话框：`opacity: 0; scale(0.95)` → `opacity: 1; scale(1)`，250ms

#### 关闭方式
- 点击背景遮罩
- 点击右上角 ×
- 点击取消/确认按钮

#### 按钮排列
- 取消按钮在左（Secondary），确认按钮在右（Primary）
- 危险操作（删除）：确认按钮为红色 `#F44336`

---

### 1.7 Input / TextField（输入框）

#### Default
- 高度 44px，内边距 12px，圆角 12px
- 背景 `#F5F5F5`，边框透明

#### Focus
- 背景变白 `#FFFFFF`，边框 `1.5px solid #8DB48E`
- 外发光 `box-shadow: 0 0 0 3px rgba(141,180,142,0.2)`

#### Error
- 边框 `1.5px solid #F44336`
- 下方显示错误文字，红色 `#F44336`，字号 12px

#### Disabled
- 背景 `#EEEEEE`，文字 `#BDBDBD`，`cursor: not-allowed`

---

### 1.8 Loading Skeleton（骨架屏）

#### 规则
- 骨架占位区域形状与实际内容一致
- 背景色：灰色块 `#E0E0E0`
- 闪烁动画：从左到右线性渐变遮罩，颜色 `#E0E0E0 → #F5F5F5 → #E0E0E0`
- 动画时长：1.5s，循环

#### 使用场景
- 页面首次加载
- 下拉刷新后等待数据

---

### 1.9 Pull-to-Refresh（下拉刷新）

#### 触发阈值
- 下拉距离 ≥ 60px 时触发刷新
- 未达阈值时，Header 跟随拖拽移动（弹性）

#### 刷新状态
- 显示自定义刷新动画（旋转的叶子图标）
- 刷新完成后，动画消失，内容回弹

---

### 1.10 Tab（标签页）

#### 样式
- 标签文字下方有 2px 指示条
- 非选中：文字 `#636E72`，无指示条
- 选中：文字 `#8DB48E`，指示条颜色 `#8DB48E`

#### 切换动画
- 指示条滑动：宽高动画 250ms ease
- 内容切换：`opacity` 淡入淡出，200ms

## 2. 页面级交互流程

### 2.1 首次进入（定位流程）

```
用户打开页面
    ↓
检测浏览器定位权限
    ↓
[有权限] → 请求定位 → 获取坐标 → 调用天气API → 展示数据
    ↓
[无权限] → 显示"定位未授权"提示 → 展示默认城市（北京）天气
    ↓
加载骨架屏 → 数据返回 → 骨架屏→内容切换动画（fadeInUp）
```

### 2.2 城市切换流程

```
点击 Header 城市名
    ↓
Bottom Sheet 城市选择页滑入
    ↓
用户搜索/选择城市
    ↓
Sheet 关闭 → Loading 骨架屏覆盖全卡片
    ↓
新城市天气数据返回 → 各卡片依次淡入切换（Hero → Forecast → Outfit → Tips）
```

### 2.3 穿着推荐交互

```
天气数据加载完成
    ↓
根据温度/天气/湿度 判断推荐规则
    ↓
推荐卡片展示 + 搭配语
    ↓
[可选] 点击"换一套"按钮 → 切换到备用推荐搭配
    ↓
卡片切换动画（fade + slide，300ms）
```

### 2.4 提示卡片交互

```
根据天气条件自动匹配提示卡片
    ↓
提示卡片依次入场（stagger 动画，间隔 100ms）
    ↓
[可交互] 点击卡片 → 展开详细信息
    ↓
展开动画（高度过渡 + 内容淡入，250ms）
```

## 3. 手势与设备适配

### 3.1 Touch 手势

| 手势 | 区域 | 行为 |
|------|------|------|
| 轻点 | 任意可点击元素 | 触发 click 效果 |
| 长按 | 天气图标 | 显示天气详情浮层（1s 后触发） |
| 拖拽 | 页面整体 | 页面垂直滚动（无侧滑返回） |
| 下拉 | 页面顶部 | 触发 Pull-to-Refresh |
| 上滑 | Bottom Sheet 内容区底部 | 关闭 Sheet |

### 3.2 键盘导航

- `Tab`：按 DOM 顺序聚焦下一个可交互元素
- `Enter / Space`：触发聚焦元素的点击行为
- `Escape`：关闭当前弹层（Bottom Sheet / Modal）
- 焦点管理：弹层打开时焦点锁定在弹层内，关闭时焦点回归触发元素

## 4. 状态定义

### 4.1 全局状态

| 状态 | 含义 |
|------|------|
| `idle` | 初始状态，未发起请求 |
| `loading` | 数据请求中，显示骨架屏 |
| `success` | 数据加载成功 |
| `error` | 数据加载失败，显示错误态 |

### 4.2 天气数据状态

| 状态 | UI 表现 |
|------|---------|
| 晴天 ☀️ | 暖黄渐变背景，太阳旋转动画 |
| 多云 ☁️ | 灰白渐变背景，云朵飘移动画 |
| 小雨 🌧️ | 淡蓝渐变背景，雨滴下落动画 |
| 大雨 🌧️ | 深蓝灰背景，雨滴快速下落 |
| 雪天 ❄️ | 浅蓝白背景，雪花飘落动画 |
| 雷暴 ⛈️ | 紫灰背景，闪电闪烁动画 |
| 加载中 | 骨架屏 + 叶子旋转动画 |
| 定位失败 | 灰白背景 + 定位失败图标 + 重试按钮 |

## 5. 错误处理

### 5.1 网络错误

- 首次加载失败 → 显示重试按钮 + 友好文案
- 刷新失败 → Toast 提示"刷新失败，请检查网络"，保留上一次数据

### 5.2 定位失败

| 原因 | 用户提示 | 兜底方案 |
|------|---------|---------|
| 浏览器不支持定位 | "您的浏览器不支持定位" | 显示默认城市 |
| 用户拒绝定位 | "定位权限未开启" + 引导开启按钮 | 显示默认城市 |
| 定位超时（>10s） | "定位超时" + 重试按钮 | 显示默认城市 |
| 定位服务不可用 | "无法获取位置信息" | 显示默认城市 |

### 5.3 数据异常

- 温度数据缺失 → 显示 "--°"
- 天气描述缺失 → 显示"天气未知"
- 城市无数据 → 提示"该城市暂不支持"，回退到默认城市

## 6. 空状态（Empty States）

### 6.1 无天气数据（初始化前）

- 中央展示：森林小鹿插画（SVG）
- 文案："正在获取天气信息…"
- 下方：叶子旋转 Loading 动画

### 6.2 无穿着推荐

- 文案："暂无推荐，看看其他信息吧～"
- 不显示空卡片区域

### 6.3 无提示卡片

- 该区块整体隐藏，不占位

---

## 7. 性能注意事项

- 骨架屏须在数据请求发出后立即展示（不要等 300ms 才显示）
- 图片资源使用 WebP 格式，雪碧图合并小图标
- 动画尽量使用 `transform` 和 `opacity`（GPU 加速）
- 页面切换时及时清理未完成的动画（避免内存泄漏）
- 禁止在滚动事件中直接操作 DOM（使用 `requestAnimationFrame` 节流）
