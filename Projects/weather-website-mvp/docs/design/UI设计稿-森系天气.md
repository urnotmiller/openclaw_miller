## 1. 页面整体结构

### 1.1 页面架构

```
┌──────────────────────────────────────────────┐
│  Header（顶部导航栏）                          │
│  [定位图标] [城市名 ▼]    [⚙设置] [👤头像]    │
├──────────────────────────────────────────────┤
│                                              │
│  Hero Weather Card（主天气卡片）              │
│  —— 展示今天当前天气 + 生活指数概览            │
│                                              │
├──────────────────────────────────────────────┤
│  3-Day Forecast Strip（三日预报横条）         │
│  [昨天] [今天 ★] [明天]                       │
├──────────────────────────────────────────────┤
│                                              │
│  Outfit Recommendation（穿着推荐区）          │
│  [天气图标] [推荐穿着标签] [搭配说明]          │
│                                              │
├──────────────────────────────────────────────┤
│                                              │
│  Tips & Reminders（温馨提示区）               │
│  [带伞] [加衣] [防晒] [补水] 等提醒卡片        │
│                                              │
└──────────────────────────────────────────────┘
```

### 1.2 视觉参考（文字描述）

整体为**森系可爱风**，灵感来源：春日森林 + 绘本插画风格。

- **主视觉色调**：浅森林绿 (#8DB48E) + 奶油白 (#FFF8F0)
- **点缀色**：珊瑚粉 (#F4A0A0)、奶茶棕 (#C4956A)、雾霾蓝 (#A8C5D9)
- **圆角**：全部使用大圆角（16px~24px border-radius），营造柔和感
- **阴影**：轻柔的暖灰色阴影，不使用锐利的黑阴影
- **图标风格**：圆润的线性图标（Line Icons），类似 Font Awesome 或 Phosphor Icons

## 4. 字体系统（Typography）

### 4.1 字体家族

```css
/* 中文优先 */
font-family: "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Noto Sans SC", sans-serif;

/* 数字 / 英文 */
font-family: "Nunito", "SF Pro Display", -apple-system, sans-serif;
```

### 4.2 字号规范

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| H1 | 28px | Bold (700) | 1.3 | 页面主标题 |
| H2 | 20px | Semibold (600) | 1.4 | 区块标题 |
| H3 | 16px | Medium (500) | 1.4 | 卡片标题 |
| Body | 14px | Regular (400) | 1.6 | 正文描述 |
| Caption | 12px | Regular (400) | 1.5 | 辅助说明、时间戳 |
| Temp (温度) | 72px | Bold (700) | 1.0 | 主卡片温度数字 |
| Number | 18px | Semibold (600) | 1.2 | 数字类标签 |

### 4.3 字重使用规范

- **Bold (700)**：温度数字、页面主标题、按钮文字
- **Semibold (600)**：卡片标题、标签文字、数值
- **Medium (500)**：城市名、次级标题
- **Regular (400)**：正文描述、提示文字

## 8. 动效规范（Motion & Animation）

### 8.1 全局动效参数

```css
--ease-default: cubic-bezier(0.4, 0, 0.2, 1);   /* 默认缓动 */
--ease-bounce:  cubic-bezier(0.68, -0.55, 0.27, 1.55); /* 弹跳缓动 */
--duration-fast: 150ms;   /* 微交互：hover、点击反馈 */
--duration-base: 300ms;  /* 常规：展开、滑入 */
--duration-slow: 500ms;  /* 大区块：页面切换 */
```

### 8.2 各场景动画

| 场景 | 动画名称 | 参数 | 描述 |
|------|---------|------|------|
| 页面进入 | fadeInUp | 0→1 opacity, 20px→0 translateY, 500ms | 整体内容从下向上淡入 |
| 卡片 hover | scaleUp | scale 1→1.02, 200ms ease | 卡片轻微放大 |
| 今天选中卡片 | glowPulse | box-shadow pulse, 2s infinite | 品牌绿光呼吸动画 |
| 下拉箭头 | rotateArrow | rotate 0→180deg, 300ms ease | 点击城市下拉时旋转 |
| 天气图标 | weatherFloat | translateY ±4px, 3s infinite ease-in-out | 天气图标轻微浮动 |
| 雨滴动画 | rainDrop | translateY 0→100vh, opacity 1→0, 1s | 雨天背景雨滴下落动画 |
| 温度数字 | countUp | 从0滚动到目标值, 1s | 数字从0滚动到实际温度 |
| Toast 提示 | slideIn | translateX 100%→0, 300ms ease | 提示从右侧滑入 |
| 页面切换 | fadeSlide | opacity + translateX, 300ms | 页面内容切换动画 |

## 9. 组件清单

### 9.1 Button（按钮）

| 变体 | 样式描述 |
|------|---------|
| Primary | 背景 #8DB48E，文字白，圆角 12px，hover 深绿 #5C8A5C |
| Secondary | 背景透明，边框 1.5px #8DB48E，文字 #8DB48E |
| Ghost | 无边框，文字 #636E72，hover 背景 #F5F5F5 |
| Icon Button | 40x40px 圆形，背景透明，hover 背景 #F0F0F0 |

**尺寸：** 小 (28px 高)、中 (36px 高)、大 (44px 高)

### 9.2 Card（卡片）

- 默认白色背景，圆角 16px，`shadow-sm`
- Hover: `shadow-md`，轻微上浮 `translateY(-2px)`
- 可选中态：左侧品牌绿 4px 竖条

### 9.3 Chip / Tag（标签）

- Pill 形状，`radius-full`
- 背景色根据语义变化（绿色=推荐、橙色=提示、灰色=次要）
- 尺寸：12px 文字，内边距 4px 12px

### 9.4 Weather Icon（天气图标）

- 尺寸系列：16px / 24px / 32px / 48px / 64px
- 风格：线性图标 + 圆润线条
- 支持 CSS 动画（旋转、浮动、闪烁）

### 9.5 Toast / Notice（提示条）

- 顶部 Fixed 展示，圆角 12px，背景白 + 左侧彩色条
- 自动 3s 消失，支持手动关闭
- 颜色对应提示类型（绿色=成功、橙色=警告、红色=错误）

### 9.6 Bottom Sheet（底部弹层）

- 从底部滑入，最大高度 70vh
- 顶部有圆形拖拽指示器（宽 36px，高 4px，灰色）
- 背景遮罩 rgba(0,0,0,0.4)

### 9.7 Loading Skeleton（骨架屏）

- 使用 `shadow-sm` 卡片占位
- 骨架色：#E0E0E0，闪烁动画（从左到右渐变遮罩，1.5s 循环）

## 10. 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| Mobile | < 640px | 单列，Header 压缩，Forecast 横向滚动 |
| Tablet | 640px ~ 1024px | 双列，部分卡片 2 列排列 |
| Desktop | > 1024px | 最大宽度 480px（居中），模拟 App 视觉 |

> **设计说明：** 本产品以移动端优先，视觉上呈现为"手机外壳内的 App"效果，在桌面端两侧留灰边，中心展示 App 界面。

## 11. 无障碍设计（Accessibility）

- 所有图标按钮需有 `aria-label`
- 颜色对比度符合 WCAG AA 标准（文字与背景对比度 ≥ 4.5:1）
- 支持键盘导航（Tab 顺序合理）
- 所有交互元素最小点击区域 44x44px
- 减少动效：支持 `prefers-reduced-motion` 媒体查询
